브라우저

1. 기능
사용자 선택 자원을 서버에 요청하고 브라우저에 표시한다.

자원의 주소는 URI(Uniform Resource Identifier)로 결정된다.
브라우저는 W3C(World Wide Web Consortium)에서 지정한 명세에 따라 HTML 파일을 해석해서 표시한다.


2. 구성요소
- 사용자 인터페이스(페이지 창 제외 나머지)
- 브라우저 엔진 -> 자료 저장소(웹데이터베이스)
- 렌더링 엔진(요청한 콘텐츠를 표시한다. HTML파일 파싱)
-- 통신(HTTP요청과 같은 네트워크를 호출한다.)
-- 자바스크립트 해석기
-- UI백엔드(플랫폼에서 명시하지 않은 일반적인 인터페이스를 말한다,)


3. 렌더링 엔진
- 역할
-- 요청 받은 내용중 먼저 받은 내용씩 브라우저 화면에 표시
- 파이어폭스-게코엔진(렌더엔진=형상엔진, 리플로)
- 사파리, 크롬-웹킷엔진(렌더엔진, 배치(Layout))
- 순서
-- 통신으로부터 요청한 문서 내용 얻기(8KB단위로)
-- HTML문서 파싱
-- 콘텐츠 트리 내부에서 태그를 DOM노드로 변환
-- 스타일 요소 파싱
-- 렌더 트리 생성(스타일 정보, HTML표시 규칙)
-- 배치(노드가 정확한 위치)
-- UI백엔드(형상 만들기)
